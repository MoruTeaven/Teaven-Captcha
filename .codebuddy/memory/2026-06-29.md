## 后台实时统计推送（SSE）实现

为 admin-dashboard.html 实现"像微信一样"的自动更新——新数据来了自动出现在面板上，不需要手动刷新。

**方案选型**：放弃轮询（D1 按行读计费、空跑浪费），改用 SSE（Server-Sent Events）。后台面板是单向展示，SSE 比 WebSocket 更轻，Cloudflare Workers 原生支持 `ReadableStream` 返回 SSE。

**后端改动**（`src/index.ts`）：
1. 新增 `GET /admin/stats/stream`：SSE 流，admin 鉴权（cookie session），每 15 秒推送一次统计快照，每 20 秒发 keepalive 注释帧防边缘回收，单连接最长 5 分钟主动关闭（`EventSource` 自动重连），监听 `request.signal.abort` 立即清理定时器。
2. 抽出 `buildAdminStatsSnapshot` 函数，聚合 today/all_time/by_provider/top_apps/active_apps，给 SSE 和普通 `GET /admin/stats` 共用。
3. `handleAdminStats` 响应结构从 `{ totals, by_provider }` 改为 `{ stats: { today, all_time, by_provider, top_apps, active_apps } }`——**接口形状有变更**，旧的消费方需适配。
4. 导出 `corsHeaders`（http.ts）供 SSE 复用。

**前端改动**（`admin-dashboard.html`）：
1. 页头加 `.live-status` 徽章（实时绿点脉冲 / 上次更新 / 已断开红点），每 15 秒刷新"X 秒前"文案。
2. `<script>` 块重写：用 `EventSource` 订阅 `/admin/stats/stream`，`onmessage` 解析快照并渲染 4 个指标卡 + 应用表格。
3. 可见性感知：`visibilitychange` 隐藏时 `es.close()` 省资源，可见时重连拿最新数据。
4. 降级兜底：SSE `onerror` 后 60 秒用一次性 `fetch('/admin/stats')` 补一次；EventSource 不支持时直接退到 30 秒轮询。
5. "刷新数据"按钮绑定 `refreshNow`（一次性 fetch + toast）。
6. 硬编码的 `apps` 数组和指标数字全部改为从快照渲染。

**验证**：`npm run typecheck` 通过；`npx wrangler deploy --dry-run` 通过（57.01 KiB）。

**文档**：DEVELOPMENT.md 9.3 节补充 `GET /admin/stats` 完整响应字段和 `GET /admin/stats/stream` SSE 协议说明；README 列表新增一条实时统计能力。

**注意事项**：
- SSE 鉴权用 cookie session（`EventSource` 不能自定义 header），跨域部署需保证同源或 cookie 透传。
- `admin-dashboard.html` 通过 `window.__TEAVEN_API_BASE__` 配置 API 基址，默认同源（空字符串）。
- D1 查询频率由服务端 15 秒间隔控制，客户端无脑接收，不浪费。
